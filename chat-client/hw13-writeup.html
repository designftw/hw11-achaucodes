Overview

The goal of my messaging app is to provide private messaging similar to iMessage, while providing a design that eliminates memory slips. It is designed for people who text family, friends, and other individuals frequently. Its features focus on eliminating slips such as forgetting to respond or missing important messages.

The key features of my app include grouping messages that haven't been responded to, pinning messages, and being able to set reminders to respond. Additionally, my app provides similar functionality that other messaging apps do, including the ability to create groupchats, reply to messages, like messages, etc.

Not Responded Category

As seen in Figure 1, my app groups conversations into three categories: "All", "Not Responded", and "Pinned". The purpose of these groups is to increase efficiency when searching for important messages, as well as to prevent memory slips by grouping all not-responded-to conversations in one place.

Reminders To Respond

Figure 2 shows the setting reminders to respond part of my interface. Reminders appear at the times they are set for in the details dropdown at the top of the page. Reminders can be set both in the 'All' and 'Not Responded' groups of conversations. There are two options for setting times to respond: "at" for "at a specific time", and "in" for "in a number of hours". The user can set reminders for multiple conversations at once due to the check-box select implementation.
